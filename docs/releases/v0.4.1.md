# Vulcan-Anvil Ex v0.4.1

Release date: 2026-05-30

`v0.4.1` is a stabilization patch for the `0.4.x` workflow. It keeps the trace-context, staged QA, dashboard, and release-pr direction from `v0.4.0`, while making worker execution safer when runs take longer than expected or when Agy returns an empty stdout.

## Highlights

- Worker execution now uses a soft timeout model for build/evidence/qa-fix runs.
- A hard timeout remains the absolute process-tree kill boundary.
- Agy/Antigravity runner results no longer depend only on stdout; transcript fallback is used when available.
- Wave completion is blocked when the Run still has open issues or a blocked/failed result state.
- Run execution records now include timeout policy and extension metadata.

## Worker Timeout

`execution.default_timeout_seconds` is now the first soft timeout checkpoint. When the checkpoint is reached, Vulcan checks for meaningful progress:

- non-heartbeat worker status changes,
- worktree git diff/status changes,
- changed file count increases,
- runner stdout/stderr/log growth during an active phase.

If progress is detected, Vulcan extends the deadline by `execution.extension_seconds`, up to `execution.max_extensions`. `execution.hard_timeout_seconds` is the final cap and terminates the process tree regardless of progress.

## Agy Transcript Fallback

Windows `agy.exe 1.0.3` can exit with code 0 while stdout remains empty. In that case, the actual model response may still be present under:

```text
%USERPROFILE%/.gemini/antigravity-cli/brain/<conversation-id>/.system_generated/logs/transcript.jsonl
```

Vulcan now extracts the conversation id from the Agy log and checks the transcript for `MODEL` responses. If a response exists, the run is not treated as `failed_empty_output` just because stdout is empty.

## Upgrade Notes

Existing projects can receive the new framework files with:

```powershell
python vulcan.py upgrade
```

Projects that already have `vulcan.config.json` should confirm the `execution` section contains:

```json
{
  "execution": {
    "default_timeout_seconds": 2400,
    "hard_timeout_seconds": 5400,
    "extension_seconds": 600,
    "max_extensions": 3,
    "progress_grace_seconds": 300
  }
}
```
