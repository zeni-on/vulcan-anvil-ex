# Vulcan-Anvil Ex v0.4.0 Release Notes

Release date: 2026-05-29

`v0.4.0`은 audit workflow 기반 AI agent coding framework를 더 단단하게 만들기 위한 실험적 minor release입니다. `v0.3.x`의 브랜치, worker, staged QA 모델은 유지하면서 trace-context 그래프, Dashboard Trace Explorer, Gate 4 실패 보고 계약, Release PR 안정화, fixture 기반 회귀 smoke 검증을 추가했습니다.

## Highlights

- `python vulcan.py trace-context` 명령을 추가했습니다. 요구사항추적표를 그래프 원장으로 사용해 seed ID 주변의 관련 ID, target contracts, source document 후보를 출력합니다.
- Dashboard 문서 Drawer에 Trace Explorer MVP를 추가했습니다. 직접 edge 그래프, ID 제목, 관련 ID 목록을 확인할 수 있습니다.
- Gate 4 `qa-execution`을 강화했습니다. QA worker는 실패를 즉시 수정하지 않고 원인, 로그, 재현 명령, 영향 ID, FIND/CR/ISSUE 후보를 보고해야 합니다.
- `release-pr` guard를 추가하고 PR body를 `.vulcan/release/release-pr-body.md`에 안정적으로 생성하도록 정리했습니다.
- fixture smoke regression에 trace-context, QA workspace 재사용, release-pr 실패 guard 검증을 추가했습니다.
- `workflow.integration_branch`는 `dev` 고정이 아니라 프로젝트별 설정 가능한 통합 브랜치임을 문서화했습니다.

## Upgrade Notes

- 기존 프로젝트는 `python vulcan.py upgrade` 후 새 workflow를 테스트합니다.
- 업그레이드 후 `vulcan.config.json.workflow.integration_branch` 값을 확인합니다.
- Gate 4 QA는 `QA-000`으로 시작하고, `QA-001`~`QA-003`은 `QA-000`이 준비한 QA workspace를 재사용합니다.
- Release PR 생성과 merge는 분리합니다. `release-pr`은 검토 단위인 PR 생성/갱신까지만 담당하고, merge는 명시 승인 뒤 수행합니다.
- Trace Explorer는 MVP입니다. 직접 관계 확인에는 유용하지만, 깊은 그래프 탐색과 검색 UX는 후속 polish 항목입니다.

## Known Follow-Ups

- 새 `v0.4.0` 기준으로 sample project를 다시 진행하고 실제 회귀를 fixture smoke에 반영합니다.
- trace-context 결과를 `run-new`와 `wave-start`의 Run 초안 품질 개선으로 연결합니다.
- Dashboard Trace Explorer는 샘플 사용 결과를 보고 필요한 탐색 기능만 후속 보강합니다.
- PR 교차검증과 dispatcher 자동화는 worker, QA, release 흐름이 더 안정된 뒤 진행합니다.
