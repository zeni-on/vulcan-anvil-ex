# Vulcan-Anvil Ex v0.4.2

Release date: 2026-05-30

`v0.4.2` is a stabilization patch for worker execution. It keeps the `v0.4.1` Agy transcript fallback and Run/Wave guards, then changes long-running worker supervision from soft-timeout extension first to a progress watchdog first model.

## Highlights

- Worker execution now uses a progress watchdog by default for `run-exec` and `agent-run --mode work`.
- `progress_probe_seconds`, `no_progress_timeout_seconds`, and `min_runtime_seconds` are added to the execution config.
- The watchdog records `active`, `quiet`, `stalled`, and `timeout_hard` states.
- Dashboard Agent details now show watchdog state, last progress, timeout policy, and timeout reason.
- The roadmap now treats worker watchdog observability as part of the `0.4.x` stabilization line.

## Worker Watchdog

The default execution policy now probes worker progress periodically:

```json
{
  "execution": {
    "progress_probe_seconds": 300,
    "no_progress_timeout_seconds": 900,
    "min_runtime_seconds": 120,
    "hard_timeout_seconds": 5400
  }
}
```

The watchdog looks for meaningful progress such as worker status changes, worktree diff/status changes, changed file count growth, and runner log growth. Heartbeat-like writes from the watchdog itself are not treated as worker progress.

If no meaningful progress is observed for the configured no-progress window after the minimum runtime, the run ends with `timeout_reason: no_progress_timeout`. The hard timeout remains the absolute cap.

## Upgrade Notes

Existing projects can receive the new framework files with:

```powershell
python vulcan.py upgrade
```

After upgrade, confirm `vulcan.config.json` contains the new `execution` watchdog fields. Older soft-timeout extension fields are still kept for compatibility and fallback behavior.
