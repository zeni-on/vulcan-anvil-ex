# 개발표준 검토 Skill

## 사용할 때

Gate 2에서 Gate 3로 넘어가기 전, 또는 Gate 4 구현 검수 시 사용한다.

구현 전에 개발표준정의서가 확정되었는지, build persona가 일관된 방식으로 코딩할 수 있는 기준이 충분한지 검토한다.

## 필수 입력

- `docs/core/TRACEABILITY_RULES.md`
- `docs/core/SECURITY_BASELINE.md`
- `docs/templates/DEVELOPMENT_STANDARD_TEMPLATE.md`
- 관련 개발표준정의서
- 관련 프로그램명세서
- 관련 테스트케이스

## 절차

1. 개발표준정의서 상태가 Draft 또는 템플릿 플레이스홀더 상태인지 확인한다.
2. 언어, 런타임, 프레임워크, DB, 테스트 도구, 빌드/패키지 방식이 확정되었는지 확인한다.
3. 패키지 구조와 계층 책임이 구현자가 바로 따를 수 있을 만큼 구체적인지 확인한다.
4. 메시지 리소스, 예외 처리, 로그, 설정값, 외부 의존성 규칙이 있는지 확인한다.
5. 주석 컨벤션과 추적 ID 표기 규칙이 있는지 확인한다.
6. 보안 구현 기준이 `SECURITY_BASELINE.md`의 관련 항목과 연결되어 있는지 확인한다.
7. 단위/통합/UI/성능 테스트 파일명, 함수명, 실행 명령, 증적 기준이 있는지 확인한다.
8. 누락이 기존 설계 범위 안이면 `FIND`, 기술스택 또는 범위 변경이면 `CR`로 분류한다.

## 출력

다음을 반환한다.

- 개발표준정의서 상태
- 확정된 기술스택과 패키지 구조
- 누락된 코딩/주석/테스트/보안 기준
- build persona가 구현 가능한지 여부
- `FIND` 또는 `CR` 권고
