# 화면 검토 Skill

## 사용할 때

Gate 2에서 Gate 3로 넘어가기 전, 또는 Gate 4 화면 구현 검수 시 사용한다.

화면이 있는 기능인데 화면 목록, 화면 상태, 와이어프레임/시안, 메시지, UI 테스트/증적 기준이 충분한지 검토한다.

## 필수 입력

- `docs/core/TRACEABILITY_RULES.md`
- `docs/core/AGENT_PERSONAS.md`
- `docs/templates/SCREEN_SPEC_TEMPLATE.md`
- 관련 요구사항정의서
- 관련 기능명세서
- 관련 화면설계서
- 관련 요구사항추적표

## 절차

1. 관련 `REQ`, `AC`, `FUNC`, `SCR`, `PGM`, `SEC` ID를 식별한다.
2. 사용자 흐름을 기준으로 필요한 화면이 모두 `SCR-ID`로 식별되었는지 확인한다.
3. 각 화면에 기본 상태, 오류 상태, 빈 상태, 권한별 상태가 필요한지 확인한다.
4. 화면설계서에 입력 항목, 버튼, 메시지 위치, 이벤트, 호출 프로그램이 있는지 확인한다.
5. 와이어프레임, 시안, HTML/CSS/JS 화면 퍼블리싱 산출물, 이미지 중 하나 이상의 화면 구조 증적이 있는지 확인한다.
6. UIREF, 화면 퍼블리싱 산출물, 외부 시안이 구현 기준이면 UI Implementation Contract가 있는지 확인한다.
7. 필수 유지 요소, 변경 허용 항목, 변경 금지 항목, 비교 방식이 구현자가 따를 수 있을 만큼 구체적인지 확인한다.
8. Gate 3에서 만들 UI 테스트와 Gate 4에서 만들 스크린샷 증적 기준이 정의되어 있는지 확인한다.
9. 누락이 기존 설계 범위 안이면 `FIND`, 요구사항 또는 화면 범위 변경이면 `CR`로 분류한다.

## 출력

다음을 반환한다.

- 검토한 `SCR-ID`
- 누락된 화면 또는 상태
- 와이어프레임/시안 존재 여부
- UI Implementation Contract 존재 여부와 준수 가능성
- UI 테스트 및 캡처 증적 기준
- `FIND` 또는 `CR` 권고
