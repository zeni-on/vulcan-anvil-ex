# 추적성 검토 Skill

## 사용할 때

요구사항, 설계, 구현, 테스트, 증적, Run 출력 간 추적성을 검증할 때 사용한다.

## 필수 입력

- `docs/core/ID_SYSTEM.md`
- `docs/core/TRACEABILITY_RULES.md`
- `docs/core/AGENT_RUN_PROTOCOL.md`
- 현재 요구사항추적표가 있으면 해당 문서
- 관련 요구사항, 설계, 테스트, 증적 문서

## 절차

1. 검토 범위와 관련 ID를 식별한다.
2. 기대되는 추적 체인을 구성한다.

```text
REQ/NREQ -> AC -> FUNC -> SCR/PGM -> DB/SEC -> UT/IT/PT/UI -> evidence
```

3. 변경된 모든 산출물에 관련 ID가 있는지 확인한다.
4. 모든 테스트가 인수기준, 보안항목, 비기능 기준 중 하나를 증명하는지 확인한다.
5. 증적 파일이 결과 문서에 참조되어 있는지 확인한다.
6. 누락 항목을 `ISSUE`, `FIND`, `CR` 중 하나로 분류한다.

## 출력

다음을 반환한다.

- 검토 범위
- 추적성 누락 항목
- 필요한 문서 갱신
- 필요한 테스트 또는 증적 갱신
- 후속 Run 필요 여부
