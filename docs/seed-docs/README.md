# 프로젝트 주입 문서

`seed-docs`는 `vulcan.py init` 또는 이후 프로젝트 생성 흐름에서 새 프로젝트로 복사하거나 참조할 기준 문서를 보관한다.

## 폴더 구분

| 경로 | 용도 | 에이전트 접근 |
| --- | --- | --- |
| `docs/seed-docs/reference-standards/` | 공공데이터 공통표준, 소프트웨어 개발보안 가이드 등 프로젝트 설계에 주입할 표준 원문 | 읽기 가능 |
| `docs/ref-docs/` | 감리 사례, 내부 참고자료, 민감하거나 프로젝트별로 공개하면 안 되는 문서 | 기본 제외 |

## 운영 원칙

- `seed-docs`의 문서는 새 프로젝트 산출물 작성 시 기준 문서로 사용할 수 있다.
- 원문 전체를 산출물에 복사하지 않고, 필요한 ID, 기준명, 요약 판단만 산출물에 남긴다.
- `docs/ref-docs/`는 민감 참고문서 영역으로 유지하며, 기본 Run 입력에서 제외한다.
