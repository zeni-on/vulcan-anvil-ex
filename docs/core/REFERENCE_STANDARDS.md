# Vulcan-Anvil Ex 참고표준 준용 규칙

> 상태: 초안 v0.1
> 목적: 설계와 문서 작성 시 공공 보안 가이드와 공공데이터 표준을 어떻게 참조하고 추적할지 정의한다.

## 1. 기본 원칙

Vulcan-Anvil Ex 산출물은 프로젝트 내부 ID만 사용하지 않고, 필요한 경우 외부 표준의 항목 ID를 함께 보존한다.

이 규칙의 목적은 다음과 같다.

- 감리 시 “어떤 기준을 준용했는가”를 명확히 설명한다.
- 보안항목이 선언에 그치지 않고 설계/구현/테스트와 연결되게 한다.
- 데이터명, 항목명, 도메인 정의가 공공데이터 표준과 충돌하지 않도록 한다.
- 외부 표준이 바뀌어도 Ex 내부 ID 체계는 안정적으로 유지한다.

## 2. 참고자료 구분

현재 기준 참고자료:

| 참조 코드 | 문서명 | 용도 |
| --- | --- | --- |
| `KISA-SD-2021` | 소프트웨어 개발보안 가이드(2021.12.29) | 분석, 설계, 구현, 테스트 단계의 개발보안 활동과 보안설계 기준 참고 |
| `KISA-SWVD-2021` | 소프트웨어 보안약점 진단가이드(2021) | 보안약점 진단 절차, 설계단계 보안항목 진단, 구현단계 보안약점 제거 기준 참고 |
| `PUBLIC-DATA-STD-2025-11` | 공공데이터 공통표준(2025.11월) | 용어, 단어, 도메인, 데이터 항목명 검토 |

표준 원문 위치:

| 구분 | 경로 | 사용 방식 |
| --- | --- | --- |
| 프로젝트 주입 표준 | `docs/seed-docs/reference-standards/` | `vulcan.py init` 또는 설계 Run에서 읽기 전용으로 참조 |
| 민감 참고문서 | `docs/ref-docs/` | 기본 제외. 사용자가 명시적으로 허용한 경우에만 참조 |

## 3. 보안 가이드 문서 관계

`소프트웨어 개발보안 가이드`와 `소프트웨어 보안약점 진단가이드`는 같은 문서가 아니다.

차이:

| 문서 | 성격 | 주 사용 시점 |
| --- | --- | --- |
| 소프트웨어 개발보안 가이드 | 개발 생명주기 전체에서 개발자가 보안활동을 수행하기 위한 가이드 | 요구사항, 설계, 구현, 테스트 계획 |
| 소프트웨어 보안약점 진단가이드 | 진단원이 보안약점과 보안설계 반영 여부를 확인하기 위한 진단 중심 가이드 | 감리, 보안진단, 검증, 조치 확인 |

다만 두 문서는 설계단계 보안설계 기준 `SR1-1` ~ `SR4-1`을 공통으로 다룬다.

실무 적용 원칙:

- 설계/개발 산출물에는 `KISA-SD-2021` 기준을 우선 참조한다.
- 감리/진단/검증 산출물에는 `KISA-SWVD-2021` 기준도 함께 참조한다.
- 같은 `SR` ID가 두 문서에 모두 있는 경우, Ex 산출물에는 `SR-ID`를 기준으로 적고 필요 시 참조 문서를 병기한다.

예시:

```text
SEC-001 비밀번호는 평문으로 저장하지 않는다.
참조 표준: KISA-SD-2021 SR2-3, KISA-SD-2021 SR2-6, KISA-SD-2021 SR2-7
진단 참조: KISA-SWVD-2021 SR2-3
```

## 4. 설계단계 보안설계 기준 ID

설계단계 보안설계 기준은 `SR` ID로 참조한다.

| SR-ID | 구분 | 항목명 | 적용 예 |
| --- | --- | --- | --- |
| `SR1-1` | 입력데이터 검증 및 표현 | DBMS 조회 및 결과 검증 | SQL 입력값 검증, 정적 쿼리, 바인딩 |
| `SR1-2` | 입력데이터 검증 및 표현 | XML 조회 및 결과 검증 | XPath/XQuery 입력값 검증 |
| `SR1-3` | 입력데이터 검증 및 표현 | 디렉토리 서비스 조회 및 결과 검증 | LDAP 입력값 검증 |
| `SR1-4` | 입력데이터 검증 및 표현 | 시스템 자원 접근 및 명령어 수행 입력값 검증 | 파일, 프로세스, 명령어 입력값 검증 |
| `SR1-5` | 입력데이터 검증 및 표현 | 웹 서비스 요청 및 결과 검증 | XSS 방지, 응답값 인코딩 |
| `SR1-6` | 입력데이터 검증 및 표현 | 웹 기반 중요 기능 수행 요청 유효성 검증 | CSRF, 중요 기능 요청 검증 |
| `SR1-7` | 입력데이터 검증 및 표현 | HTTP 프로토콜 유효성 검증 | URL, Header, Redirect 검증 |
| `SR1-8` | 입력데이터 검증 및 표현 | 허용된 범위 내 메모리 접근 | 메모리 접근 범위 검증 |
| `SR1-9` | 입력데이터 검증 및 표현 | 보안기능 입력값 검증 | 권한, 인증, 보안결정 입력값 검증 |
| `SR1-10` | 입력데이터 검증 및 표현 | 업로드·다운로드 파일 검증 | 파일 형식, 크기, 경로, 무결성 검증 |
| `SR2-1` | 보안기능 | 인증 대상 및 방식 | 중요 기능 인증 적용, 인증 우회 방지 |
| `SR2-2` | 보안기능 | 인증 수행 제한 | 로그인 시도 횟수 제한, 실패 이력 로깅 |
| `SR2-3` | 보안기능 | 비밀번호 관리 | 비밀번호 생성규칙, 저장, 변경 정책 |
| `SR2-4` | 보안기능 | 중요자원 접근통제 | 중요 기능/자원 권한 검증 |
| `SR2-5` | 보안기능 | 암호키 관리 | 암호키 생성, 보관, 접근, 파기 |
| `SR2-6` | 보안기능 | 암호연산 | 안전한 알고리즘, 해시, 솔트, 난수 |
| `SR2-7` | 보안기능 | 중요정보 저장 | 개인정보, 비밀번호 등 중요정보 저장 보호 |
| `SR2-8` | 보안기능 | 중요정보 전송 | 인증정보, 개인정보, 쿠키 전송 보호 |
| `SR3-1` | 에러처리 | 예외처리 | 오류 메시지 정보노출 방지, 안전한 예외 처리 |
| `SR4-1` | 세션통제 | 세션통제 | 세션 ID 보호, 세션 간 데이터 공유 방지 |

## 5. Ex 보안항목 작성 방식

`SEC` 항목은 내부 보안 ID와 외부 참조 표준을 모두 가진다.

권장 형식:

```yaml
security_id: SEC-001
name: 비밀번호 평문 저장 금지
description: 비밀번호는 안전한 해시 알고리즘과 솔트를 사용해 저장한다.
applies_to:
  - PGM-001
  - DB-001
reference_standards:
  - KISA-SD-2021 SR2-3
  - KISA-SD-2021 SR2-6
  - KISA-SD-2021 SR2-7
verification:
  - UT-001
  - IT-001
```

규칙:

- `SEC-ID`는 Ex 내부 추적 ID다.
- `SR-ID`는 외부 보안 가이드의 준용 기준이다.
- 하나의 `SEC` 항목은 여러 `SR` 기준을 참조할 수 있다.
- 하나의 `SR` 기준은 여러 `SEC` 항목에 적용될 수 있다.
- 보안 민감 `SEC` 항목은 화면/프로그램/DB/인터페이스 중 하나 이상의 적용 대상과 연결해야 한다.
- 보안 민감 `SEC` 항목은 하나 이상의 테스트 또는 리뷰 증적과 연결해야 한다.

## 6. 공공데이터 공통표준 참조 방식

공공데이터 공통표준은 데이터명, 화면 항목명, DB 컬럼명, API 필드명 검토에 활용한다.

참조 대상:

| 시트 | 활용 |
| --- | --- |
| 공통표준용어 | 데이터 항목명과 영문 약어명 검토 |
| 공통표준단어 | 단어 분해와 영문 약어 검토 |
| 공통표준도메인 | 데이터 타입, 길이, 표현 형식 검토 |

권장 형식:

```yaml
data_field: user_name
label_ko: 사용자명
public_data_standard:
  term: 사용자명
  english_abbreviation: USER_NM
  domain: 명V100
  reference: PUBLIC-DATA-STD-2025-11
```

규칙:

- 공공/행정 데이터 성격이 강한 프로젝트는 데이터명세서에서 공공데이터 표준 참조를 우선 검토한다.
- 표준 용어와 다른 명칭을 써야 할 경우 사유를 남긴다.
- 화면 표시명, API 필드명, DB 컬럼명이 서로 다를 수 있으므로 매핑을 명시한다.
- 개인정보, 인증정보, 식별정보는 보안항목 `SEC`와도 연결한다.

| 산출물 | 반영 위치 |
| --- | --- |
| 요구사항정의서 | 비기능 요구사항, 보안 및 개인정보 고려사항 |
| 요구사항추적표 | `SEC-ID`, `참조 표준`, `증적` 컬럼 |
| 기능명세서 | 보안 고려사항, 입력/출력 항목, 관련 화면/프로그램/테스트 |
| 화면설계서 | 입력항목 검증, 화면 이벤트, 보안항목 |
| 프로그램명세서 | 입력값 검증, 인증/인가, 암호화, 예외처리, 세션처리 |
| DB명세서 | 데이터 항목명, 도메인, 개인정보/중요정보 여부 |
| 테스트케이스 | `AC`, `SEC`, `SR` 기준별 검증 케이스 |
