## 1. 문서 목적

본 문서는 프로젝트 이해관계자, 사용자군, 업무 범위, 제외 범위, 승인 지점을 정의한다.

범위 정의는 요구사항 확정과 변경요청 판단의 기준선으로 사용한다.

## 2. 작성 기준

- 이해관계자는 역할 기준으로 작성하고, 실명은 필요한 경우에만 기록한다.
- 범위는 포함 범위와 제외 범위를 함께 작성한다.
- 제외 범위는 나중에 변경요청 또는 백로그 후보가 될 수 있으므로 근거를 남긴다.
- 승인자와 검토자는 문서 승인뿐 아니라 Gate 통과 판단의 책임을 가진다.
