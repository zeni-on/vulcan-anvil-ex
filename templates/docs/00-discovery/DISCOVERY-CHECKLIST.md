# Phase 0: Discovery 완료 체크리스트

Phase 0의 산출물이 Gate 1 진입에 충분한 수준인지 판단하는 기준.
**모든 항목이 충족되어야 Gate 1로 전환할 수 있다.**

## 필수 항목

### 용어 (Glossary)
- [ ] `glossary/glossary.md`에 프로젝트 핵심 용어가 정의되어 있다
- [ ] 이해관계자 간 용어 해석 차이가 없다

### 상위 요구사항 (Requirements)
- [ ] 기능 요구사항(F-xx)이 도출되어 있다
- [ ] 비기능 요구사항(N-xx)이 도출되어 있다
- [ ] 각 요구사항에 출처(법령, ISP, 이해관계자 등)가 명시되어 있다
- [ ] 신규/고도화 구분이 되어 있다

### 기능 상세 (Functional Detail)
- [ ] 주요 기능의 입력/처리/출력/예외가 정의되어 있다
- [ ] 기능 간 의존성이 식별되어 있다

### 인프라 설계 (Infrastructure)
- [ ] 네트워크/Zone 아키텍처가 설계되어 있다
- [ ] 서버 구성이 정의되어 있다
- [ ] 외부 시스템 연동 방식이 정의되어 있다

### 기술 검토 (Technical Review)
- [ ] 주요 기술 선택에 대한 검토서가 작성되어 있다
- [ ] 대안 비교 및 선택 근거가 명시되어 있다

### FP/공수 산정 (Estimation)
- [ ] IFPUG 기반 FP가 산정되어 있다
- [ ] 총 공수, 일정, 인력이 추정되어 있다

## 선택 항목

- [ ] 인프라 요구사항(I-xx)이 도출되어 있다 (인프라가 있는 프로젝트)
- [ ] 보안 아키텍처가 설계되어 있다 (보안 요구가 높은 프로젝트)
- [ ] 법령/규정 준수 사항이 정리되어 있다 (규제 대상 프로젝트)

### 고도화 프로젝트 전용 (기존 시스템이 있는 경우)

- [ ] 현행 코드베이스 구조가 분석되어 있다 (`audit/codebase-analysis.md`)
- [ ] 공개 API 목록과 Breaking change 후보가 식별되어 있다 (`audit/api-surface.md`)
- [ ] 외부 의존성과 보안 취약점이 점검되어 있다 (`audit/dependency-audit.md`)
- [ ] 마이그레이션 전략이 수립되어 있다 (`audit/migration-strategy-vN.md`)

## 미확정 사항 허용 기준

Phase 0은 탐색 단계이므로 일부 미확정 사항이 있을 수 있다.
단, 아래 조건을 만족해야 한다:

- 미확정 사항이 문서에 "미확정: [사유]"로 명시되어 있다
- 미확정 사항이 Gate 1(REQ/AC 정의)을 진행하는 데 차단 요소가 아니다
- 미확정 사항의 확인 대상(누구에게 물어야 하는지)이 식별되어 있다

## Gate 1 전환 절차

1. 이 체크리스트의 필수 항목을 검토한다
2. 사용자의 승인을 받는다
3. BA가 `docs/00-discovery/requirements/` 최신 버전을 기반으로 Gate 1의 REQ-NNN/AC-NNN 형식으로 변환을 시작한다
