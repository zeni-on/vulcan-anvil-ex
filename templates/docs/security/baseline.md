# 보안 기준선 (Security Baseline)

> 프로젝트에 적용할 최소 보안 요구사항입니다.
> 프로젝트별 추가 보안 요건은 `compliance/` 디렉토리에 작성합니다.

## OWASP Top 10 체크리스트

| # | 카테고리 | 점검 항목 | 상태 |
|---|---------|----------|------|
| A01 | 접근 제어 | 모든 엔드포인트 인증/인가 확인 | [ ] |
| A02 | 암호화 | HTTPS 강제, 비밀번호 해싱(bcrypt) | [ ] |
| A03 | 인젝션 | SQL 파라미터 바인딩, XSS 이스케이핑 | [ ] |
| A04 | 안전하지 않은 설계 | Rate Limiting, 비즈니스 로직 남용 방지 | [ ] |
| A05 | 보안 설정 | 디버그 모드 OFF, 보안 헤더 설정 | [ ] |
| A06 | 취약한 컴포넌트 | 의존성 CVE 점검 | [ ] |
| A07 | 인증 실패 | 로그인 실패 제한, 세션 관리 | [ ] |
| A08 | 데이터 무결성 | 역직렬화 검증 | [ ] |
| A09 | 로깅 부족 | 보안 이벤트 로깅, 민감정보 미포함 | [ ] |
| A10 | SSRF | URL 화이트리스트, 내부 IP 차단 | [ ] |

## 환경 변수 보안

- 민감 정보(DB 비밀번호, API 키)는 반드시 환경변수로 관리
- `.env` 파일은 `.gitignore`에 포함
- 코드에 하드코딩된 자격증명 금지

## 프로젝트별 추가 보안 요건

`docs/security/compliance/` 디렉토리에 프로젝트별 보안 문서를 추가하면,
이 baseline.md보다 우선 적용됩니다.
